以下のアプリケーションの設計を行って。

==========

1. プロジェクト概要

- プロジェクト名: Content Reviewer（コンテンツレビューアー）
- 目的: SNS発信コンテンツの事前チェックを自動化し、広報チームの負荷を軽減しつつ、品質と正確性を担保する

## 背景

- 日本の政党が利用する内部ツール
- 議員が増えたことにより発信（YouTube, X, noteなど）の分量が飛躍的に増加
- 注目度の上昇により、不適切な表現をなるべく避けたい状態
- 現在は広報チームが全件目視レビューしており、負荷が限界

## 解決策

LLMを活用したコンテンツ自動チェックシステムを構築し、以下の観点で文章を自動検証する:

- 一般的な事実のチェック - Web検索による一般的な事実確認
- ナレッジコンテンツとの整合性確認 - note記事、マニフェストなどのドキュメントの文章をRAG検索して（不整合な部分をチェック）
- 表現ルール適合 - ナレッジコンテンツとして登録するまでもない、小さな表現のルールを登録し、該当チェックできる（Aとは表現せずBと表現する など）
- 炎上リスク判定 - 一般常識に照らし、誰かを傷付ける、貶める発言がないかのチェック
- 文章クオリティチェック - 同じ動詞の繰り返しや、シンプルな誤字脱字のチェック

入ってくるコンテンツは、最大30000字程度。
LLMを使って、AIがうまく処理できる段落ごとに分割し、それぞれに並列で上記のチェックをかける。

## 機能

- A. Slackで sns-checker アカウントにメンションがあったときにそれをフックして、そのスレッドに検証結果を返す（検証結果パーマリンクつき）
- B. Webでも文章を入力してチェックできる
- C. 過去の文章チェック一覧を確認できる
- D. WebにはGoogleログインできる（開発環境ではスキップできるように作ること）
- E. 表現ルールの管理
- F. ナレッジの手動登録・編集
- G. 記事コンテンツの自動取込（noteについて、アカウントを指定して記事URL一覧を読み込み、ボタンを推すとDB保存できる）

## Web画面イメージ
- 主にPCから利用する
- 左側にサイドバーで右側にメインコンテンツ
- サイドバーには「チェック一覧」「表現ルール」「ナレッジ」のナビゲーション、ログインユーザー名、ログアウトボタンを常時表示

### ログイン `/login`
- Google アカウントでログイン（`team-mir.ai` ドメイン限定）
- ログイン後はチェック一覧へリダイレクト
- 開発環境では `SKIP_AUTH=true` で認証スキップ用のダミーログインボタンを表示
- ドメイン制限エラーや認証エラーはクエリパラメータで画面に表示

### チェック一覧（ダッシュボード） `/`
- 過去のチェック結果を一覧テーブルで表示（タイトル・投稿先・実行者・チェック日時）
- 期間（直近7日 / 直近30日 / 全期間）、プラットフォーム（Web / Slack）、キーワードで絞り込み
- ページネーション対応
- 「新規チェック」ボタンで `/checks/new` へ遷移
- 行クリックで `/checks/{id}` へ遷移

### 新規チェック `/checks/new`
- タイトル（任意）、投稿先（YouTube / X / note / その他、任意）を入力
- 本文テキストエリア（最大30,000文字、文字数カウンター付き）
- 「チェック開始」で AI チェックを実行し、結果ページ `/checks/{id}` へ遷移

### チェック結果 `/checks/[id]`
- チェック進行中は、セクションごとの完了状況をプログレスバーで表示し、3秒間隔で自動ポーリング
- 2分以上経過した場合は「未完了セクションをチェックする」「全セクションを再チェックする」のリトライ選択肢を提示
- チェック完了後は、左カラム（55%）にセクション本文、右カラム（45%）に指摘カードを並べた2カラムレイアウト
- 指摘カードには以下を表示:
  - 観点バッジ（表現 / リスク / 品質 / 事実 / ナレッジ）
  - 重要度バッジ（注意 / 要修正）
  - 該当箇所の引用テキスト
  - 指摘メッセージと修正案
  - 適用ルールへのリンク（`/rules?highlight={ruleId}` へ遷移）
  - 「解決」トグルボタン
- セクション単位での再チェック機能
- リンクコピー（パーマリンク共有）
- チェック結果の削除（確認ダイアログ付き、削除後は `/` へ遷移）

### 表現ルール `/rules`
- ルール一覧をテーブル表示（NG表現 → OK表現のプレビュー付き）
- ルールの追加・編集（ダイアログ）・削除（確認ダイアログ）
- 有効/無効の切り替えトグル（無効ルールは半透明表示）
- チェック結果画面から `?highlight={ruleId}` で遷移した場合、該当ルール行をハイライト表示

### ナレッジ一覧 `/knowledge`
- ナレッジをテーブル表示（タイトル・ソース種別・取込日）
- ソース種別（全ソース / note / マニフェスト / 手動）とキーワードで絞り込み
- 「ソースから取り込む」で `/knowledge/sources` へ遷移
- 「手動追加」でナレッジを直接登録（ダイアログ）
- タイトルクリックで `/knowledge/{id}` へ遷移

### ナレッジ詳細 `/knowledge/[id]`
- タイトル、ソース種別、取込日、元URL（外部リンク）を表示
- ナレッジ本文の全文表示
- 編集（ダイアログ）・削除（確認ダイアログ、削除後は `/knowledge` へ遷移）

### ソース接続管理 `/knowledge/sources`
- 登録済みソースをカード形式で一覧表示（ソース種別・名前・URL・取込済み件数・未取込件数・最終取込日）
- 「新規ソース追加」で note アカウントを登録（ダイアログ）
- カードクリックで `/knowledge/sources/{id}` へ遷移

### ソース詳細 `/knowledge/sources/[id]`
- ソース名・URL・種別を表示
- 未取込セクション: チェックボックスで個別/一括選択し、「選択した N 件を取り込む」でナレッジに登録
- 取込済みセクション: ナレッジ化済みの記事・ファイルを一覧表示
- note ソースの場合は「記事一覧を再取得」で API から最新記事リストを同期


## 技術要素

- embedding: OpenAI
- 認証: SupabaseのGoogle認証（client sdkは利用しないで極力バックエンド認証に寄せてほしい）
