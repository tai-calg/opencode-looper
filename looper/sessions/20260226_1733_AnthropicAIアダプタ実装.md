## 設計判断

- 既存の OpenAIEmbeddingAdapter / StubEmbeddingAdapter パターンに従い、AIGateway interface を implements するアダプタを作成
- AI SDK v6 では `maxSteps` が `stopWhen: stepCountIs(n)` に変更されていたため、設計ドキュメントから修正して対応
- StubAIAdapter は `schema.parse({ issues: [] })` で空の指摘リストを返す設計

## 詰まった点

- AI SDK v6 で `maxSteps` パラメータが廃止され `stopWhen: stepCountIs(5)` に変更されていた。TypeScript の型チェックで検出し修正

## 次のタスクへの申し送り

- AnthropicAIAdapter の `generateTextWithWebSearch` で `stopWhen: stepCountIs(5)` を使用（設計ドキュメントの `maxSteps: 5` を SDK v6 API に適合させた）
- Composition Root で AIGateway の DI 登録が必要（ステップ F で対応予定）
