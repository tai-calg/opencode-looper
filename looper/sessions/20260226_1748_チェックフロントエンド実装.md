## 設計判断
- Server Component で初期データ取得 → Client Component に渡すパターン（SSR + ポーリング）
- API Route はポーリング専用（Server Component の loader はクライアントから呼べないため）
- `let check: CheckDetail` で明示的型注釈（biome noImplicitAnyLet 対応）
