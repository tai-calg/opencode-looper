## 設計判断
1. **DATABASE_URL ポート問題**: `supabase status` が 54322 を表示するが Docker コンテナは 55322 でリッスン。実際のポートを使用
2. **ダミーユーザー ID**: `'dummy-user-id'` は PostgreSQL UUID 型に挿入不可 → `'00000000-0000-0000-0000-000000000001'` に統一し、DB にもこの ID でユーザーを事前挿入
3. **Server→Client シリアライズ**: Next.js はドメインモデルクラス（private constructor）をプレーンオブジェクトとして渡せない → ローダー層で DTO 変換
4. **OpenAI API キー**: E2E 環境では dummy キーのため `NoopEmbeddingGateway`（ゼロベクトル返却）でバイパス
5. **削除ダイアログ strict mode 問題**: ダイアログにタイトルテキストを表示していたため、削除後のチェック時に 2 要素が strict mode 違反 → タイトルテキストを汎用メッセージに変更・削除前にダイアログを閉じる
