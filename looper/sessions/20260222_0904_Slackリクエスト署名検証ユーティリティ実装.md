# タスク: slack-signature-verification

## 内容
Slack リクエスト署名検証ユーティリティの実装。

## 作成・変更ファイル

- `apps/content-reviewer/src/lib/slack-signature.ts` **新規作成**
  - `verifySlackSignature(signingSecret, signature, timestamp, rawBody): boolean` を実装
  - node:crypto の `createHmac('sha256', signingSecret)` で `v0:{timestamp}:{rawBody}` を署名
  - `timingSafeEqual` でタイミング攻撃対策した比較
  - タイムスタンプが5分以上古い（または未来）場合は false を返す
  - 署名長が異なる場合も false（timingSafeEqual は同じ長さでないとエラーになるため）

- `apps/content-reviewer/src/lib/env.ts` **更新**
  - `getSlackSigningSecret(): string`（`SLACK_SIGNING_SECRET` 環境変数）を追加
  - `getSlackBotToken(): string`（`SLACK_BOT_TOKEN` 環境変数）を追加

- `apps/content-reviewer/test/unit/lib/slack-signature.test.ts` **新規作成**
  - 7テストケース: 正常系、タイムスタンプ期限切れ、未来タイムスタンプ、不正署名、長さ不一致、署名シークレット不一致、5分境界値

- `apps/content-reviewer/test/unit/lib/env.test.ts` **更新**
  - `getSlackSigningSecret` と `getSlackBotToken` のテストを追加（各2ケース）

## 設計判断

- `timingSafeEqual` は同じバイト長でないと TypeError になるため、長さが異なる場合は事前に false を返す
- タイムスタンプ検証で `Math.abs(now - ts) > 5 * 60` を使い、リプレイ攻撃と時刻ズレ両方を防ぐ
- 既存の env.ts パターン（環境変数なければ throw）を踏襲

## 次のタスクへの申し送り

- `verifySlackSignature` は Slack Webhook エンドポイント（API Route）で使用される想定
- 環境変数 `SLACK_SIGNING_SECRET` / `SLACK_BOT_TOKEN` は `.env.local` への追加が必要
