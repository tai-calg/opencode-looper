# タスク: fix-content-check-dummy-user-fk-violation

## 内容
E2E テスト checks.spec.ts のフローテストが外部キー制約違反で失敗していた問題を修正。

## 変更ファイル
- `apps/content-reviewer/src/app/api/checks/execute/route.ts`

## 修正内容
- `NODE_ENV=development` 時に `createUserId(DUMMY_USER_ID)` でダミーUUIDをセットしていた箇所を削除
- `userId` を `undefined` のまま `execute()` に渡すよう変更（`ContentCheck.userId` は nullable のため DB に null で保存される）
- 不要になった `DUMMY_USER_ID` 定数を削除

## 設計判断
- `ContentCheck.userId` は `userId?: UserId` と nullable な設計になっているため、開発環境では undefined を渡すだけでよい
- 本番環境（`NODE_ENV !== 'development'`）では引き続き Supabase Auth からユーザーIDを取得する

## 次のタスクへの申し送り
- この修正により E2E テスト checks.spec.ts のフローが正常に動作するはず
- 本番環境ではユーザー認証が必須（未ログインは 401 を返す）という挙動は変わらない
