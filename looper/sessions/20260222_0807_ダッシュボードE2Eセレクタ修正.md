# タスク: fix-dashboard-e2e-selectors

## 内容
`e2e/dashboard.spec.ts` で4件のテストが strict mode violation / pointer event intercept で失敗していた問題を修正。

## 変更したファイル
- `apps/content-reviewer/e2e/dashboard.spec.ts`

## 修正内容

### (1) 「新規チェック」リンク (strict mode violation)
- **問題**: `page.getByRole('link', { name: '新規チェック' })` がサイドバーと main 領域の2要素にマッチ
- **修正**: `page.getByRole('main').getByRole('link', { name: '新規チェック' })` に変更（2箇所）

### (2) フィルタUIクリック (pointer event intercept)
- **問題**: `page.getByLabel('ソース種別')` / `page.getByLabel('ステータス')` クリック時にサイドバーがインターセプト
- **修正**: `page.getByRole('main').getByLabel('ソース種別')` / `page.getByRole('main').getByLabel('ステータス')` に変更（4箇所）

## 設計判断
`page.getByRole('main')` で main 領域に絞り込むアプローチを採用。
タスク説明にある `id='filter-source'` ロケータより、セマンティックな `getByRole('main')` チェーンの方が保守性が高いため選択。
