# 20260222_0816 フィルタ Select ポインタインターセプト修正

## タスク ID
fix-filter-select-pointer-intercept

## 内容
E2E テスト dashboard.spec.ts の2件（ソース種別フィルタ変更・全てに戻すテスト）が失敗する問題を修正。

## 変更ファイル
- `apps/content-reviewer/e2e/dashboard.spec.ts`
  - フィルタ操作テスト3件の `sourceTrigger.click()` / `statusTrigger.click()` に `{ force: true }` を追加

## 設計判断
shadcn/ui の Select はネイティブ `<select>` ではなくカスタム UI コンポーネントであり、サイドバーの `div[data-sidebar='content']` 要素が CSS 上でメインコンテンツ領域に重なりポインタイベントをインターセプトしていた。

`{ force: true }` オプションは Playwright の「要素の actionability チェック」をスキップしてクリックを強制実行するもので、オーバーラップした要素を無視してターゲット要素を直接クリックできる。shadcn/ui の SelectTrigger は正しく `id="filter-source"` / `id="filter-status"` を持ち `aria-label` も設定されているため、要素自体の特定には問題なく、クリックの物理的な阻害のみが問題だった。

## 詰まった点・解決方法
特になし。根本原因が明確だったため `force: true` を追加するだけで解決。

## 次のタスクへの申し送り
- SelectTrigger へのクリック3箇所（ソース種別変更・ステータス変更・全てに戻す）全てに `force: true` を付与済み
- `pnpm verify`（lint + typecheck + build + unit test）は全件 pass を確認済み
