## タスク ID
execute-content-check-sse

## 内容
`src/app/api/checks/execute/route.ts` を新規作成。POST API Route でコンテンツチェックを実行し、SSE ストリームで進捗イベントを返す。

## 作成・変更したファイル
- `apps/content-reviewer/src/app/api/checks/execute/route.ts` (新規作成)

## 設計判断
- `architecture.md` では「API Route は外部 Webhook 等の例外のみ」とあるが、タスク指示で明示的に API Route として指定されているため従う。SSE ストリームは ReadableStream + text/event-stream で実現
- `NODE_ENV=development` 時はダミーユーザー ID（全ゼロ UUID）を使用、本番は `createSupabaseServerClient()` でセッションから userId を取得
- `onProgress` コールバックで受け取った ProgressEvent を `data: ${JSON.stringify(event)}\n\n` 形式で SSE エンコード
- エラー発生時は catch して error イベントを送出し、ストリームを正常終了（controller.close()）
- Content-Type: text/event-stream, Cache-Control: no-cache, Connection: keep-alive を設定

## 詰まった点・解決方法
- Biome の import sort エラー（createUserId を createSupabaseServerClient より前に並べる）を修正して lint 通過
