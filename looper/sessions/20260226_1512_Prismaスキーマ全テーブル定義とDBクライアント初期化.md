# prisma-schema-and-db: Prisma スキーマ全テーブル定義 + DB クライアント初期化

## 作成・変更したファイル
- `apps/webapp/prisma/schema.prisma` — 既存の空スキーマに全8テーブル定義を追加（User, Check, CheckSection, CheckIssue, ExpressionRule, KnowledgeItem, Source, SourceArticle）
- `apps/webapp/src/backend/contexts/shared/infrastructure/db/prisma-client.ts` — 新規作成。Next.js Hot Reload 対策のシングルトンパターン

## 設計判断
- 設計ドキュメントの Prisma スキーマ定義をそのまま採用。`@@map` によるスネークケーステーブル名、`@db.Uuid`/`@db.Timestamptz` による PostgreSQL ネイティブ型指定
- KnowledgeItem の `embedding` フィールドは `Unsupported("vector(1536)")` を使用（pgvector 用、将来 `$queryRaw` で検索）
- prisma-client.ts は biome のインデントスタイル（タブ）に合わせた

## 詰まった点
- 特になし。`prisma generate` は `Unsupported` 型を含むスキーマでも正常動作

## 次のタスクへの申し送り
- `prisma generate` 実行済み。`@prisma/client` の型が生成されているため、他タスクで import 可能
- DB マイグレーションは未実行（Supabase セットアップ後に `prisma db push` で反映予定）
