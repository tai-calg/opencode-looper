# Milestone 1: 開発サーバー起動・開発用ログイン・サイドバー付きダッシュボード表示

## 概要

ゼロからmonorepo（pnpm workspace + Next.js App Router）をスキャフォールディングし、SKIP_AUTH=true での開発用ログイン、サイドバー付きダッシュボードページを表示できる状態にする。`pnpm verify`（lint → prisma generate → typecheck → build → unit test）が通ることをゴールとする。

## タスク一覧

| ステップ | タスクID | Wave | 概要 |
|----------|----------|------|------|
| A | monorepo-scaffolding | 1 | monorepo + Next.js + Tailwind + shadcn/ui + 品質ツール設定 |
| B | prisma-schema-and-db | 2 | Prisma スキーマ全テーブル定義 + DB クライアント初期化 |
| C | backend-shared-and-auth | 2 | 共有ドメインモデル（Result, UserId, Timestamp）+ 開発用認証バックエンド |
| D | frontend-login-and-sidebar | 2 | ログインページ + サイドバーナビゲーションコンポーネント |
| E | auth-layout-and-pages | 3 | 認証レイアウト（サイドバー + セッション検証）+ ダッシュボード・ルール・ナレッジページ |

#### shadcn/ui コンポーネントのインストール

`apps/webapp/` ディレクトリで以下を実行:

```bash
npx shadcn@latest init --yes
npx shadcn@latest add button card input textarea badge separator avatar dialog select switch label table dropdown-menu tooltip sheet
```

これにより `src/frontend/components/ui/` 配下にコンポーネントファイルが生成される。

> **注意**: shadcn/ui init が tailwind.config.ts, globals.css, utils.ts を上書き生成する場合がある。生成後に上記の仕様と整合するよう調整すること。特に `aliases` の設定（components.json）が `@/frontend/components/ui` を指していることを確認する。

#### テストディレクトリの作成

```bash
mkdir -p apps/webapp/test/unit apps/webapp/test/integration apps/webapp/test/e2e apps/webapp/test/fixtures
```

vitest が空ディレクトリで正常終了（テスト 0 件 → 成功）することを確認する。

#### 実装パターン

- ゼロからの scaffolding。`pnpm init` → 手動で workspace 設定 → `pnpm install` の順。
- shadcn/ui は `npx shadcn@latest init` で初期化した後、必要なコンポーネントを add する。
- Tailwind CSS v4 系を使用する場合は `@tailwindcss/postcss` プラグインを使い、`tailwind.config.ts` の代わりに `globals.css` 内の `@theme` ディレクティブで設定する。shadcn/ui init の出力に従う。

#### 注意事項

- `pnpm install` 実行後、`pnpm-lock.yaml` が生成される。これはコミット対象。
- `.gitignore` に `apps/webapp/.next/` が含まれていることを確認（既に `.next/` がルートの .gitignore にある）。
- biome.json の `files.ignore` に `apps/webapp/src/frontend/components/ui` を含め、shadcn/ui 生成コードを lint 対象外にする。
- **タスク完了時に `pnpm verify` が通ることを確認する。** この時点では prisma/schema.prisma が存在しないため、verify スクリプトの `prisma generate` ステップをスキップするか、空の schema.prisma を置くこと。最低限 lint → typecheck → build が通ればよい。

- `Unsupported` 型を含むスキーマでも `prisma generate` は正常に動作する。

```typescript
export type Result<T, E = string> =
  | { success: true; value: T }
  | { success: false; error: E };

export const Result = {
  ok: <T>(value: T): Result<T, never> => ({ success: true, value }),
  err: <E>(error: E): Result<never, E> => ({ success: false, error }),
};
```

```typescript
// user-id.model.ts
export class UserId {
  private constructor(private readonly value: string) {}

  static create(value: string): UserId {
    if (!value) throw new Error('UserId cannot be empty');
    return new UserId(value);
  }

  static reconstruct(value: string): UserId {
    return new UserId(value);
  }

  toString(): string {
    return this.value;
  }

  equals(other: UserId): boolean {
    return this.value === other.value;
  }
}
```

```typescript
// auth.composition.ts
// Composition Root: 認証関連のファクトリ関数をエクスポート。
// Milestone 1 では Cookie ベースの開発用セッションのみ。
// Milestone 7 で Supabase Auth に差し替え。
import { cookies } from 'next/headers';

const DEV_SESSION_COOKIE = 'dev-session';

type DevSession = {
  userId: string;
  email: string;
  name: string;
  avatarUrl?: string;
};

export function createAuthService() {
  return {
    async getSession(): Promise<DevSession | null> {
      const cookieStore = await cookies();
      const raw = cookieStore.get(DEV_SESSION_COOKIE)?.value;
      if (!raw) return null;
      try {
        return JSON.parse(raw) as DevSession;
      } catch {
        return null;
      }
    },

    async setDevSession(session: DevSession): Promise<void> {
      const cookieStore = await cookies();
      cookieStore.set(DEV_SESSION_COOKIE, JSON.stringify(session), {
        httpOnly: true,
        secure: process.env.NODE_ENV === 'production',
        sameSite: 'lax',
        path: '/',
        maxAge: 60 * 60 * 24 * 7, // 7日間
      });
    },

    async clearSession(): Promise<void> {
      const cookieStore = await cookies();
      cookieStore.delete(DEV_SESSION_COOKIE);
    },
  };
}
```
